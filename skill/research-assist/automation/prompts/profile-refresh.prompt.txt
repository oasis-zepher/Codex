# Profile Refresh Prompt

You are running inside the `research-assist` skill repository.

## Mission

Read the current Zotero library and produce one compact machine-readable research-interest profile for the current arXiv retrieval pipeline.

## Required References

- Existing example shape: `profiles/research-interest.example.json`
- Target path for the live profile: `{{TARGET_PROFILE_PATH}}`
- Skill contract: `SKILL.md`
- Workflow reference: `references/workflow.md`
- Contract reference: `references/contracts.md`
- Zotero MCP reference: `references/zotero-mcp.md`

## Required Operating Rules

- use the bundled `research-assist-zotero-mcp` tools for live Zotero access
- call `zotero_status` first if configuration is unclear
- use `zotero_profile_evidence` as the primary evidence collection step
- use Zotero as the primary evidence source
- maintain a compact, retrieval-friendly profile
- prefer 3-10 precise interest slices over a large noisy list
- keep each interest slice arXiv-query-friendly
- preserve short method labels instead of broad topic dumping
- do not write to Zotero
- do not call `zotero_write_profile`; the controller script will persist the final JSON
- do not output Markdown prose
- do not include comments in JSON
- the first non-whitespace character of the final answer must be `{`
- the last non-whitespace character of the final answer must be `}`
- do not wrap the JSON in code fences
- do not prepend or append any explanation, note, heading, or apology
- if you are uncertain, still return the best valid JSON object that fits the contract

## Required JSON Fields

Return **JSON only** with these top-level keys:

- `schema_version`
- `profile_id`
- `profile_name`
- `updated_at`
- `maintainer`
- `zotero_basis`
- `retrieval_defaults`
- `interests`

Each `interest` must contain:

- `interest_id`
- `label`
- `enabled`
- `categories`
- `method_keywords`
- `query_aliases`
- `exclude_keywords`
- `logic`
- `notes`

## Output Gate

Your response will be rejected unless it is a single valid JSON object with no extra text before or after it.
