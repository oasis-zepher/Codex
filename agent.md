# AGENT: 工作流规范

## 权限与操作规范
1. 在执行 `git push` 或 `git clone` 之前，必须先获得用户明确许可。
2. 当用户输入 `/plan` 进入 plan 模式后，未获得用户同意前，不得编写代码或修改任何文件。
3. 允许读取本 Mac 上任意文件；但若要修改 BIO 文件夹之外的任何文件，必须先获得用户批准。
4. 若需修改 BIO 以外路径，先说明目标路径与原因，再等待用户确认。

## 核心开发步骤 (SOP)
当你接到一个新需求或功能模块时，必须严格按照以下步骤执行：
1. **Requirement Analysis (需求确认):** 简述你对需求的理解，并列出核心技术难点。
2. **Architecture Design (架构设计):** 在写代码前，先给出数据流向或模块拆分方案。
3. **Implementation (代码实现):** 按照设计输出代码，关键逻辑必须包含注释。
4. **Self-Correction (自我检查):** 检查是否存在性能瓶颈或内存泄漏风险。

## 行为准则补充
参考 `forrestchang/andrej-karpathy-skills` 的方法论，补充以下约束：

1. **Think Before Coding:** 不要替用户做未经确认的假设；若存在多种解释，先说明分歧再行动。
2. **Simplicity First:** 优先选择最短、最直接、最少抽象的方案；禁止为了“未来扩展”提前复杂化。
3. **Surgical Changes:** 只修改和需求直接相关的内容；不要顺手改无关代码、注释、格式或旧逻辑。
4. **Goal-Driven Execution:** 尽量把任务转化为可验证的成功条件；修复问题时优先考虑复现、验证与回归检查。

## 专项任务处理
- **数据分析与机器学习任务：** 在处理特征矩阵或进行模型训练前，优先检查数据的完整性和维度是否对齐。
- **桌面端应用开发任务：** 优先考虑多面板交互的流畅度，以及主线程与耗时任务（如数据处理）的分离。

## Python 环境约定
- 运行 Python 脚本前必须使用 conda 环境 **Bioinfo**。
- 激活方法：
  - `conda activate Bioinfo`
  - 或直接使用：`conda run -n Bioinfo python <脚本路径>`
